# 钢琴识谱启蒙游戏 MVP 计划

## 目标

为 4–6 岁钢琴启蒙儿童制作一个 iPad 横屏 H5 游戏：孩子通过“气球 + 发射音符”的方式，把音名或琴键和五线谱位置建立联系。

## MVP 范围

- 两种玩法并存：
  - 唱谱模式：孩子说出当前音符的音名，MVP 先用可替换的模拟关键词识别接口承载流程。
  - 弹谱模式：孩子弹出对应钢琴键，MVP 先用中央 C 校准流程和可替换的模拟音高输入承载流程。
- 单首内置自创 C 调单音练习曲。
- 两种模式各自的开场校准、本地缓存、重置入口。
- 气球升起，命中后发射音符戳破气球，音符飞回五线谱位置。
- 错误输入会做温和标记，不惩罚儿童。
- 全曲完成后按原节奏回放一遍。
- 面向 iPad Safari 横屏，触控目标大，布局适配 4:3。

## 不进 MVP

- 真实 MFCC / DTW 童声音名识别训练。
- 真实 YIN / CREPE 声学钢琴音高检测。
- 摄像头指法识别。
- 拍照 OMR。
- MIDI 键盘输入。
- 多曲库、大谱表、高低声部。

## 架构

- 继续使用无构建步骤的原生 HTML / CSS / JavaScript。
- `src/notationGame.js` 负责可测试的游戏状态、谱面、校准、命中判定和回放队列。
- `src/storage.js` 负责本地缓存封装，优先 localStorage，IndexedDB 作为后续真实发音模板扩展点。
- `src/main.js` 负责浏览器 UI、Canvas 动画、触控输入、麦克风授权入口和模拟识别按钮。
- Canvas 绘制五线谱、气球、发射音符和归位动画；DOM 承载模式选择、校准、HUD 与操作按钮。

## 验收标准

- `npm test` 通过。
- `npm run smoke` 通过并生成截图。
- 页面首屏是 iPad 横屏可玩的游戏界面，不是营销页。
- 能在唱谱 / 弹谱两种模式间切换。
- 两种模式都有校准流程、校准状态和重置入口。
- 命中当前气球后，状态进入 burst / placed，分数增加，五线谱上出现归位音符。
- 错误输入会标记尝试次数，但不终止游戏。
- 完成内置曲后进入回放状态。
- 工作记录写入 `.logs/`，并保留 `progress.md` 方便下一轮接续。
